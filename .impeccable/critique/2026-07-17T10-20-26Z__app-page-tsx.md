## New Priority Issues

### P1 — ConfirmDialog shows "0 pull requests to create" before scan

- **What**: `ConfirmDialog.tsx:59` shows `groupCount={prs.length || 0}` — before any scan, this reads "0 pull requests to create"
- **Why it matters**: Confusing. Scan hasn't happened yet. User doesn't know what "0" refers to.
- **Fix**: Show generic message when no scan completed: "Review results before creating pull requests"
- **Suggested command**: `/impeccable clarify`

### P1 — Empty state lists 5 violation types (chunking)

- **What**: `Dashboard.tsx:49` previews 5 violation types in empty state
- **Why it matters**: Exceeds working memory limit of 4 items.
- **Fix**: Drop least common type → 4 items.
- **Suggested command**: `/impeccable distill`

### P2 — Em-dash opacity-30 in dashboard empty state

- **What**: Dashboard empty state score dashes use `opacity-30` on `--color-muted`, effective ~2.8:1 contrast.
- **Why it matters**: R1 had same pattern on placeholder. We fixed the placeholder but the dash pattern is identical.
- **Fix**: Remove opacity-30 on empty state dashes. Use full `--color-muted` (6.2:1).
- **Suggested command**: `/impeccable audit`

### P2 — `# how it works` prefix

- **What**: `HowItWorksSection.tsx:50` shows `# how it works` — hash prefix feels self-consciously coded.
- **Why it matters**: Tries too hard to look "code-y." Adds noise without value.
- **Fix**: Remove `#` → "how it works"
- **Suggested command**: `/impeccable clarify`

### P2 — Button loading label always says "Scanning…"

- **What**: Button shows "Scanning…" during prioritizing and fixing phases too.
- **Why it matters**: Inaccurate for later phases. Minor but undermines status visibility.
- **Fix**: Accept phase name as prop, or use generic "Working…"
- **Suggested command**: `/impeccable clarify`

### P2 — `$` span lacks `aria-hidden`

- **What**: `ScanForm.tsx:25` decorative `$` prompt span not hidden from screen readers.
- **Why it matters**: Accessibility tool should catch this. Screen reader may announce "$" without context.
- **Fix**: Add `aria-hidden="true"`
- **Suggested command**: `/impeccable audit`

## Strengths

1. **State management completeness**: idle → scanning → prioritizing → fixing → done, each with distinct indicators. Skeleton, stepper, spinner, error, empty — no gaps.
2. **Accessibility-aware implementation**: `aria-busy`, `aria-selected`, `aria-current="step"`, `role="tablist"`, `role="dialog"`, `aria-modal`, reduced-motion detection, keyboard tab nav.
3. **Token-based design system**: All 17 components use shared CSS vars. No hardcoded colors. Consistent bezel card language throughout.
